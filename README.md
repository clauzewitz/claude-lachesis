# claude-lachesis — Claude Code 사용량 추정 메뉴 막대 앱

`~/.claude/projects/`의 대화 기록만 읽어서, Claude Code의
**현재 세션 · 이번 주(전체) · 이번 주(Sonnet)** 사용률을
메뉴 막대에서 퍼센트 막대로 보여 주는 macOS 앱입니다.

- 아무것도 설치할 필요가 없습니다 (상태줄 스크립트 불필요)
- 네트워크 접근이 전혀 없습니다 → 약관 걱정 없음
- 메뉴 막대에 현재 세션 퍼센트가 실시간으로 표시됩니다
- 소진 속도를 계산해 "이 속도면 몇 시에 한도 도달"을 예측합니다
- 초기화 시각과 함께 남은 시간("2시간 13분 남음")을 보여줍니다
- 각 카드에 집계 기준점("…부터 집계")을 표시해 계산 근거를 투명하게 보여줍니다
- 70% / 90% 도달 시 macOS 알림을 보냅니다 (같은 구간에서는 한 번만)

> ⚠️ **추정치입니다.** Anthropic이 한도(분모)를 공개하지 않으므로,
> 이 앱의 퍼센트는 공식 /usage 화면과 다를 수 있습니다.
> "대략 어느 정도 썼는지" 감을 잡는 용도로 보시면 됩니다.

## 이름의 유래

라케시스(Lachesis)는 그리스 신화 속 운명의 세 여신(모이라이) 중
**각자에게 주어진 몫의 실 길이를 재는** 여신입니다. 정해진 할당량을
헤아려 가늠하는 이 앱의 역할과 같습니다. 천리안의 파수꾼
claude-lynceus(상태줄)가 '보는 눈'이라면, claude-lachesis는 '재는 손'입니다.

## 준비물

- macOS 13 (Ventura) 이상 (앱이 **실행**되는 환경)
- **Xcode 16 이상** (빌드용. 프로젝트가 XcodeGen objectVersion 77 형식이라 필요)
- macOS 26(Tahoe)에서 실행하면 카드·배지·버튼이 리퀴드 글래스로
  표시됩니다 (글래스 효과를 컴파일하려면 Xcode 26 필요).
  macOS 13~15에서는 기존 평면 디자인으로 자동 전환됩니다.

## 실행 방법

> 🛠 **소스(git)에서 받은 경우** `Lachesis.xcodeproj`가 없습니다 — 이 프로젝트는
> `project.yml`(XcodeGen)에서 프로젝트를 생성하기 때문입니다. 먼저 아래를 한 번
> 실행하세요(이후 소스를 추가/변경할 때도 다시 실행). 릴리스 zip에는 생성된
> `Lachesis.xcodeproj`가 들어 있어 이 단계가 필요 없습니다.
>
> ```bash
> brew install xcodegen   # 최초 1회
> xcodegen generate       # project.yml → Lachesis.xcodeproj
> ```

1. `claude-lachesis.zip`의 압축을 풉니다. 아래 구조가 그대로 유지되어야 합니다.

   ```
   claude-lachesis/
   ├── Lachesis.xcodeproj/   ← 이것을 엽니다
   └── Lachesis/             ← 소스와 아이콘 (프로젝트 옆에 있어야 함)
   ```

2. `Lachesis.xcodeproj`를 더블클릭해 Xcode로 엽니다.
3. ⌘R (Run)을 누릅니다.
4. 메뉴 막대에 게이지 아이콘과 퍼센트(예: `31%`)가 나타납니다. 클릭하면 펼쳐집니다.

> 서명 관련 창이 뜨면 Signing & Capabilities 탭에서 본인 Apple ID 팀을
> 선택하시면 됩니다. (로컬 실행용 서명으로도 동작하도록 설정되어 있습니다)

## 정확도를 높이는 두 가지 설정 (⚙️ 설정 버튼)

1. **주간 초기화 시각**: Claude Code에서 `/usage`로 확인한 주간 초기화
   요일·시각(예: 월요일 오전 10시)을 입력하면, 이번 주 사용량을 그 시각
   기준으로 정확히 자릅니다. 직접 설정을 끄면 기본값(월요일 오전 10시)
   기준으로 계산합니다.
2. **한도 기준**:
   - **자동 추정**(기본): 지금까지 관찰된 최대 사용량을 100%로 봅니다.
     한도에 닿아 본 적이 있어야 의미 있는 값이 나옵니다.
   - **직접 입력**: 한도에 닿았을 때 카드에 표시된 토큰 수를 적어 두면
     점점 정확해집니다. (단위: 백만 토큰)

## 동작 원리

Claude Code는 모든 대화를 `~/.claude/projects/*.jsonl`에 기록하며,
각 응답마다 토큰 수(입력·출력·캐시 생성·캐시 읽기)가 들어 있습니다.
이 앱은 1분마다 그 파일을 읽어:

- **세션**: 첫 메시지 시각부터 5시간을 한 구간으로 묶어 현재 구간의
  토큰 합계와 초기화 시각(시작+5시간)을 계산합니다.
- **주간**: 매주 초기화 시각(직접 설정 또는 기본값 월요일 오전 10시) 기준으로
  토큰 합계를 계산하고, Sonnet 모델만 따로 집계합니다.
- **퍼센트**: 위 합계 ÷ 한도(자동 추정 또는 직접 입력) 입니다.

토큰 종류·모델별 가중치가 비공개라 공식 수치와 차이가 나는 것은
이 방식의 본질적 한계입니다.

## 폴더 구조

```
claude-lachesis/
├── project.yml                   # XcodeGen 프로젝트 정의 (진실원본)
├── Lachesis.xcodeproj/           # project.yml 에서 생성 (git 미추적)
├── make_icon.py                  # 앱 아이콘 재생성 스크립트 (선택)
└── Lachesis/
    ├── LachesisApp.swift         # 진입점 · 메뉴 막대 라벨(실시간 %)
    ├── ContentView.swift         # 메인 화면 · 사용량 카드 · 캡슐 막대
    ├── SettingsView.swift        # 설정 (한도 기준 · 주간 초기화 시각)
    ├── UsageStore.swift          # 설정 보관 · 1분 주기 갱신
    ├── UsageEngine.swift         # JSONL 스캔 · 구간/주간 계산 · 한도 추정
    └── Assets.xcassets/AppIcon   # 앱 아이콘 (다크 슬레이트 + 사용량 막대)
```
